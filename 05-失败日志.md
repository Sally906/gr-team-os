# 五、失败日志（Failure Log）

**目的**：不是追责，而是防止同样的错误发生两次。

## 模板

| 字段 | 说明 |
| :--- | :--- |
| 失败编号 | FL-YYYY-MM-DD-001 |
| 登记人 | 当事DRI |
| 发生日期 | |
| 关联流程/项目 | 如“高企申报” |
| 错误描述 | 客观描述发生了什么 |
| 实际影响 | 时间、金钱、关系、合规 |
| 根本原因 | 流程缺失？信息过时？判断失误？ |
| 即时解决方案 | 怎么解决的 |
| 预防措施 | 修改哪个SOP或知识库 |
| 关联更新 | 更新了哪个条目 |
| 负责人 | 谁执行预防措施 |
| 状态 | 进行中/已完成/已关闭 |

## 使用规则

- 任何失败须在 **48小时内** 录入
- 每月召开 **失败复盘会**，分析Top类型
- KPI 中包含“失败日志完成率”和“预防措施落地率”
